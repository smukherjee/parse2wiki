## Infographic: BAGERA Global View Architecture

BAGERA is a baggage handling system powered by RESA. The architecture shows data flow from input sources through the core BRS Engine to output systems.

### Input Sources (Left)
- **infopax** (AODB) or any other AODB
- **bagway** - Other Baggage Messages Sources (BagLink, Bag Message, BUM)
- **on board** (DCS powered by RESA)
- **crews** (BMS)
- Other Local Systems

### Core Processing - baguera BRS (Center)
- **BRS Engine** handles:
  - Arrival/Departure Flights
  - Bags Info
  - Tag Reprint, Runway Follow-up, Alerts, Arrival Tracking
  - Administration (Tracking, Authorization to load)
  - Aircraft Load Strategy
  - Storage Device/Types
  - Supervision, Remote Control

- **BAGERA CLIENT** (Multi-language):
  - Scanners (WiFi, 4G) + ring/gloves
  - ATR Automatic Tag Readers (arches)
  - RFID integration
  - Worldwide Tracking

### Output Systems (Right)
- **clever** (Post Exploitation History)
- **paxtrack** (PAX Tracking by RESA)
- **bagway** (Other Baggage Destination Systems: BagLink, BagMessage, DCS)
- Other Systems (BHS, BRS, Drop-off Systems)
- Worldwide Tracking System

### Communication Protocols

**Input to BAGUERA (Inbound Messages):**
- `infopax` (AODB) → Bag Messages
- `bagway` → BagLink, BagMessage, BUM
- `on board` (DCS) → DCS Messages
- `crews` (BMS) → BMS Messages
- Other Local Systems → Local Protocols

**BAGUERA to Output (Outbound Messages):**
- To `clever` → BPM, BSM (Post Exploitation History)
- To `paxtrack` → PAX Data
- To `bagway` (Destination) → BagLink, BagMessage, DCS
- To Other Systems → BHS/BRS Protocols
- To Worldwide Tracking → Tracking Data

**BAGUERA ↔ CLIENT (Bidirectional):**
- BPM (Baggage Processing Message)
- BagMessage
- BSM (Baggage Source Message)

**Legend:**
